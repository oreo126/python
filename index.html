git add 🍡

把變更加入暫存區（Stage），準備被記錄。

展開語法與範例 
git add .            # 加入所有變更
git add README.md     # 加入指定檔案
git add src/          # 加入資料夾
  • 用途：選擇要進入下一次 commit 的檔案。
  • 小提醒：用 git status 看目前哪些檔案已 staged。

git commit 🧁

把暫存區的內容記錄成一次提交（快照）。

展開語法與範例 
git commit -m "feat: 新增會員登入功能"
git commit -m "fix: 修正資料驗證錯誤"
git commit -am "chore: 調整 linter 設定"  # -a 同時 add 已追蹤檔案
  • 訊息撰寫：清楚敘述「做了什麼」與「為什麼」。
  • 慣例：可用 feat、fix、docs 等前綴。

git push 🍬

把本地提交推送到遠端（如 GitHub）。

展開語法與範例 
git push origin main
git push -u origin main   # 設定追蹤，之後可直接 git push
  • 先決條件：需已設定遠端 origin 與分支。
  • 衝突：若遠端有你本地沒有的提交，可能需要先 git pull。

git pull 🍓

抓取遠端更新並合併到本地。

展開語法與範例 
git pull origin main
git pull --rebase origin main   # 以 rebase 整理歷史
  • 差異：pull = fetch + merge（或 rebase）。
  • 團隊習慣：有些團隊偏好 --rebase 以維持線性歷史。

git revert 🍮

建立一個「反向提交」，用新的提交抵銷指定提交的變更。

展開語法與範例 
git log --oneline            # 找到要還原的 commit
git revert <commit-hash>      # 產生一個反向提交
git revert HEAD~2             # 還原倒數第 3 個提交
  • 優點：保留歷史，不會改動既有提交。
  • 注意：與 reset 不同，revert 適合公開分支。

git checkout 🍪

切換分支或回到某個提交版本；也可檢出檔案。

展開語法與範例 
git checkout -b feature/login   # 建立並切換到新分支
git checkout main               # 切換到 main
git checkout <commit-hash>       # 進入分離 HEAD 狀態（僅瀏覽）
  • 現代用法：推薦使用 git switch（切分支）與 git restore（還原檔案）。
  • 安全：分離 HEAD 狀態下的修改若要保留，請建立分支。

git merge 🍯

把另一個分支的變更合併到目前分支。

展開語法與範例 
git checkout main
git merge feature/login       # 合併 feature/login 到 main
# 若發生衝突，修正檔案後：
git add .
git commit                    # 完成合併提交
  • 合併策略：預設為 recursive；可視情況改 fast-forward 或 no-ff。
  • 替代：用 rebase 可得到更線性的歷史，但需注意共享分支風險。
